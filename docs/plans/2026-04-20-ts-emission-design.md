# Epic 0.6 — TypeScript Emission Design

**Date:** 2026-04-20
**Status:** Approved
**Epic:** 0.6
**Depends on:** 0.5 (Name Resolution)

---

## Goal

Typed AST to runnable TypeScript. Effect annotations and contracts are erased; structural types map 1:1. Output verified by tsc. Ships before effect checking so there's an executable feedback loop from the start.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Emission library | ts-morph | Rich API, built-in .d.ts generation, future-proof |
| Sum type mapping | Discriminated unions + factory functions | Uniform `{ kind, ...fields }` shape is agent-friendly, JSON-serializable, greppable |
| Extern blocks | Emit as TS `import` statements | Output must be runnable; erasing externs would break execution |
| Output directory | `--outdir` flag, default `dist/` | Keeps source clean, matches TS conventions |
| tsc verification | Built into `radahn build` via ts-morph `getPreEmitDiagnostics()` | Accuracy first; catches emission bugs immediately |
| Unit type | Renamed from `Unit` to `Void` | Familiar to agents from TS training data; reduces friction |

## Emission Pipeline

```
.rd source -> Lexer -> Parser -> Resolver -> Emitter -> .ts + .d.ts
```

### Entry Point

`src/emit/ts-emitter.ts` exports:

```ts
function emit(
  file: AstNode,
  arena: Arena<AstNode>,
  resolutions: Map<NodeId, NodeId>
): EmitResult

type EmitResult = {
  ts: string
  dts: string
  diagnostics: Diagnostic[]
}
```

Walks the AST via `switch` on `node.kind` with `exhaustive()`. Uses ts-morph `Project` to build an in-memory source file, then serializes to string. The `.d.ts` is generated by ts-morph's declaration emit.

## Type Mapping

| Radahn | TypeScript |
|--------|-----------|
| `Int` | `number` |
| `Float` | `number` |
| `String` | `string` |
| `Bool` | `boolean` |
| `Void` | `void` |
| `List[T]` | `T[]` |
| `T?` | `T \| null` |
| `(A, B, C)` | `[A, B, C]` |
| `{ x: Int, y: Int }` | `{ x: number; y: number }` |
| `fn(A, B): C` | `(arg0: A, arg1: B) => C` |
| Generic `T` | `T` (pass-through) |
| `Int where x > 0` | `number` (refinement erased) |

### Sum Types

Given `type Result[T, E] = Ok(T) | Err(E)`, emit three things:

1. Variant interface: `interface Ok<T, E> { readonly kind: "Ok"; value_0: T }`
2. Union alias: `type Result<T, E> = Ok<T, E> | Err<T, E>`
3. Factory function: `function Ok<T, E>(value_0: T): Result<T, E> { return { kind: "Ok", value_0 } }`

### Erasure

These produce no TS output:
- Effect annotations (`! Fs, Net`)
- Contracts (`@pre`, `@post`, `@cost`)
- Module metadata (`version`, `summary`, etc.)

## Statement & Expression Mapping

### Functions

```
fn add(a: Int, b: Int): Int    ->  function add(a: number, b: number): number {
  a + b                         ->    return a + b;
end                             ->  }
```

Last expression in a block is implicit return — emitter wraps in `return`.

### Let Bindings

```
let x: Int = 5                 ->  const x: number = 5
let mut y: Int = 5             ->  let y: number = 5
```

### If Expressions

```
if cond then a else b          ->  cond ? a : b          (expression context)
if cond                        ->  if (cond) {            (statement context)
  doThing()                    ->    doThing();
end                            ->  }
```

### Match Expressions

```
match result                   ->  (() => {
  Ok(v) => v                   ->    if (result.kind === "Ok") { const v = result.value_0; return v; }
  Err(e) => 0                  ->    if (result.kind === "Err") { const e = result.value_0; return 0; }
end                            ->  })()
```

IIFE with chained `if` checks on the `kind` discriminant.

### Extern Blocks

```
extern "fs" {                  ->  import { readFile } from "fs";
  fn readFile(path: String): String ! Fs
}
```

Effect annotations erased. Each extern fn becomes a named import.

### Operators

| Radahn | TypeScript |
|--------|-----------|
| `++` (concat) | `+` |
| `==` / `!=` | `===` / `!==` |
| All others | Pass-through |

### Visibility

`pub` functions/types emit with `export` keyword. Non-pub omit it.

### Numeric Literals

Underscored separators (`0xFF_00`) are stripped for TS compatibility.

## Build Command

`radahn build [files...] --outdir dist/`

1. Read `.rd` files from arguments
2. For each file: lex -> parse -> resolve -> emit
3. Write `<outdir>/<name>.ts` and `<outdir>/<name>.d.ts`
4. Run ts-morph `getPreEmitDiagnostics()` to verify correctness
5. Report diagnostics (JSON or human-readable via `--json`)
6. Exit 0 on success, 1 on errors

## Testing Strategy

### Layer 1: Unit Tests (`tests/emitter.test.ts`)

Feed AST nodes to the emitter, assert on TS string output. Coverage:
- Functions (params, return types, implicit return)
- Let bindings (const vs let for mut)
- All type mappings (primitives, generics, tuples, records, fn types)
- Sum types (interfaces + union + factories)
- If expressions (ternary vs statement)
- Match expressions (IIFE + kind checks)
- Extern -> import
- Operator mappings (`++` -> `+`, `==` -> `===`)
- `pub` -> `export`
- Numeric literal normalization
- Void literal -> `undefined`
- Refinement erasure (emit base type)
- Contract/effect erasure (no output)
- BlockExpr -> IIFE
- NamedArg -> positional

### Layer 2: Conformance + Snapshots

Run emitter on all existing fixtures (HELLO, REFUND, RESULT, REFINEMENT, LET_AND_BINOP, EXTERN_FS, CONTRACTS). Verify:
- ts-morph type checker passes
- Snapshot output for regression detection

### Layer 3: E2E Tests (`tests/e2e-build.test.ts`)

4 scenarios:
1. `console.log("hello from radahn")` via extern — basic pipeline
2. Sum type construction + match + correct return value
3. Generic function instantiation with concrete types
4. Declaration ordering (forward references)

### Layer 4: Error Handling

- Parser error-recovery nodes don't crash the emitter
- Intentionally broken output triggers `getPreEmitDiagnostics()`
